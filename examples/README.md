# OpenTime Examples

Concrete `.ot` examples to help you understand and adopt the format in your own tools.

Each file is valid OpenTime v0.2 YAML — clone the repo, open them in your editor, and experiment with your parser.

| File | Description |
|------|-------------|
| [minimal-tasks.ot](./minimal-tasks.ot) | A minimal file with just tasks — good starting point for simple tools |
| [daily-plan.ot](./daily-plan.ot) | A basic day plan mixing tasks, events, and reminders |
| [novel-project.ot](./novel-project.ot) | A project container linking goals, tasks, and events |
| [extensions.ot](./extensions.ot) | App-specific `x_*` fields that survive round-trips |
| [sample.ot](./sample.ot) | Comprehensive example exercising all seven item types |

For the full format definition, see the [specification](../spec/core.md).
